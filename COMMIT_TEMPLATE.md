# Conventional Commit Template

type(scope?): message court et informatif

# Exemples de types :
# feat     : ajout de fonctionnalité (feature)
# fix      : correction de bug
# test     : ajout/modif de tests
# docs     : changement de documentation
# refactor : refonte ou amélioration interne
# chore    : maintenance (build, dépendances, etc.)
# style    : changements de formatage (pas de logique métier)

# Format:
# - Ligne de titre ≤ 72 caractères
# - Phrase explicite (en anglais)
# - Ref interne ou externe si pertinent (#42)
# - Pas de point (.) à la fin de la ligne de titre
